# Architecture Decision Records (ADRs)

We record every significant technical decision so that future contributors (including future-us) understand **why** things are the way they are.

## Format

Each ADR follows: **Context** (what's the situation) → **Options** (what we considered) → **Decision** (what we chose) → **Consequences** (tradeoffs accepted).

Status: `Proposed` → `Accepted` → `Superseded` (if replaced by a later ADR).
