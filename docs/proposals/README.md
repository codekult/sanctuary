# Proposals

Feature proposals are written **before implementation** to force clear thinking and document intent.

## Format

Each proposal should include:

- **Goal**: What are we trying to achieve?
- **Scope**: What's included and explicitly excluded?
- **Design**: Data model changes, UI sketches, API endpoints
- **Open questions**: Unresolved decisions (resolve before implementing)
- **Phases**: If the feature is large, how to deliver incrementally
